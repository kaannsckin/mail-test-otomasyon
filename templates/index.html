🏗 Sistem Genel Bakış

Mail Otomasyon Test Sistemi, Safir Posta sunucu geçiş projesi kapsamında farklı mail sunucuları (EMS, Gmail, Outlook) ve istemciler (iOS, Android, Outlook Desktop) arasındaki mesaj iletim kalitesini otomatik olarak test eden bir araçtır.

Sistem şu bileşenlerden oluşur:

BileşenDosyaAçıklama
Web Arayüzüapp.pyFlask tabanlı kontrol paneli (bu ekran). Konfigürasyon, test başlatma, canlı log izleme ve raporlama.
Orkestratörmain.pyTest senaryolarını CSV'den okur, sırasıyla çalıştırır. Gönderim → Alım → Analiz akışını yönetir.
Göndericisender.pySMTP üzerinden 5 farklı senaryo tipinde mesaj gönderir. Çoklu ek dosya desteği.
Alıcıreceiver.pyIMAP polling ile gönderilen mesajı bulur, MIME yapısını ayrıştırır.
Analizcianalyzer.pyClaude API ile gönderilen/alınan mesajları karşılaştırır, PASS/FAIL kararı verir.
Mesaj Şablonlarımessage_templates.pySenaryo bazlı kısa/orta/uzun Türkçe mesaj taslakları.
CSV Parsercsv_parser.pyTest checklist CSV'sini okuyarak kombinasyon ve senaryo yapısını oluşturur.
Raporlayıcıreporter.pyHTML görsel rapor ve CSV sonuç dosyası üretir.
2FA Yöneticisiauth_manager.pyTOTP otomatik üretim ve interaktif MFA kod girişi akışı.

🚀 Hızlı Başlangıç (5 Adım)

Ön Koşullar: Python 3.10+, pip, internet bağlantısı (Claude API için) ve en az bir mail hesabı (EMS, Gmail veya Outlook).
  1. Bağımlılıkları kur: pip install -r requirements.txt
  2. Config dosyasını oluştur: cp config.yaml.example config.yaml
  3. Web arayüzünü başlat: python app.py → Tarayıcıda http://localhost:5000 aç
  4. Konfigürasyon sekmesinden sunucu bilgilerini ve Claude API key'ini gir, Kaydet'e bas
  5. Test Çalıştır sekmesinden kombinasyon seç ve ▶ Testi Başlat'a tıkla
İpucu: Hızlı test için sağ üstteki "▶ Hızlı Başlat" butonu tüm kombinasyonları tek tıkla çalıştırır.

Gmail App Password Nasıl Alınır?

  1. Google Hesabı → Güvenlik → 2 Adımlı Doğrulama'yı etkinleştir
  2. Güvenlik → 2 Adımlı Doğrulama → sayfanın altında "Uygulama Şifreleri" bölümü
  3. Uygulama: Posta, Cihaz: Diğer → "Mail Otomasyon" yaz → Oluştur
  4. Verilen 16 haneli şifreyi Konfigürasyon → Gmail → App Password alanına yapıştır

Claude API Key Nasıl Alınır?

  1. console.anthropic.com adresine giriş yap
  2. API Keys → Create Key → İsim ver → Kopyala
  3. Konfigürasyon sekmesinde Claude API → API Key alanına yapıştır
Güvenlik: config.yaml dosyası .gitignore'da tanımlıdır. Şifreler ve API key'ler repoya gitmez. Yalnızca config.yaml.example commit edilir.

TOTP Otomatik Üretim (Önerilen)

Authenticator uygulamanızdaki Base32 secret değerini Güvenlik & 2FA sekmesine kaydettiğinizde, her test çalışmasında 6 haneli kod otomatik üretilir — test duraklamaz.

Secret'ı nereden bulurum? Google Authenticator / Authy'ye hesap eklerken QR kod yerine "Manuel giriş" seçeneğini seçin. Gösterilen Base32 metin (ör. JBSWY3DPEHPK3PXP) secret'tır. Bunu kopyalayıp yapıştırın.

Manuel Kod Girişi

TOTP secret kaydetmezseniz, test çalışırken 2FA gereken sunucuda ekranınıza modal pencere açılır. Authenticator uygulamanızdaki kodu girin → test devam eder.

📝 Mesaj Şablonları (Kısa / Orta / Uzun)

Her senaryo için 3 farklı uzunlukta gerçekçi Türkçe iş yazışması şablonu tanımlıdır. Kombinasyon indeksine göre otomatik rotasyon yapılır:

  • Kombinasyon 0 → Kısa mesaj (1-2 satır, hızlı bilgilendirme)
  • Kombinasyon 1 → Orta mesaj (1 paragraf + maddeler)
  • Kombinasyon 2 → Uzun mesaj (çok bölümlü teknik rapor)
  • Kombinasyon 3 → Tekrar kısa... (döngüsel)
SenaryoKısaOrtaUzun
Plain TextTek satır bilgilendirmeHaftalık durum, 4 madde4 bölümlü teknik rapor
Ek Dosya"Ekte bulunur" notuBroşür + gereksinim açıklamasıDosya listesi + MIME kontrol detayı

Neden farklı uzunluklar? Kısa mesajlarda header oranı yüksektir (encoding testi), uzun mesajlarda gövde bütünlüğü ve karakter korunması test edilir. Bu kombinasyon gerçek dünya kullanımını simüle eder.

Tüm şablonlarda Türkçe özel karakter doğrulama satırı bulunur: ğüşıöçĞÜŞİÖÇ. Ayrıca her mesaj sonuna standart kurumsal imza bloğu (Bilgi Teknolojileri Müdürlüğü) eklenir.

📎 Ek Dosyalar (Attachments)

Ek Dosya (attachment) senaryosunda attachments/ klasöründeki gerçek dosyalar kullanılır. Her test çalışmasında 3 dosya birlikte eklenir:

Detaylı Akış

  1. CSV Parse: mail_test_checklist.csv dosyasından kombinasyon ve senaryo bilgileri okunur
  2. Şablon Seçimi: Kombinasyon indeksine göre kısa/orta/uzun mesaj şablonu seçilir
  3. SMTP Gönderim: Senaryo tipine uygun MIME yapısı oluşturulur (plain, multipart/mixed, multipart/related, vb.) ve gönderilir
  4. IMAP Polling: Alıcı sunucuda mesajın ulaşması beklenir (varsayılan 15sn, 3 deneme, 5sn aralıkla)
  5. MIME Ayrıştırma: Alınan mesajın headers, parts, attachments ve inline_images bilgileri parse edilir
  6. Claude Analizi: Gönderilen metadata + alınan içerik Claude API'ye gönderilerek senaryo tipine özel kontrol noktaları değerlendirilir
  7. Raporlama: JSON sonuçlar HTML rapor ve CSV dosyasına dönüştürülür

▶ Test Çalıştırma Adımları

Web Arayüzünden (Önerilen)

  1. Test Çalıştır sekmesine git
  2. Kombinasyon Seçimi: Test etmek istediğin kombinasyon kartlarına tıkla (birden fazla seçilebilir). "Tümünü Seç" butonu hepsini seçer.
  3. Senaryo Tipi: Hepsini test etmek için "Tümü" seçili kalsın. Belirli bir senaryo için ilgili pill'e tıkla.
  4. Mod Seçimi:
    • Normal: Mail gönderir, alır, analiz eder — tam test
    • Dry Run: Sadece SMTP bağlantısını test eder, mail göndermez
  5. ▶ Testi Başlat'a tıkla — otomatik olarak Canlı Loglar sekmesine geçilir
  6. Test tamamlanınca sonuçlar Sonuçlar ve Raporlar sekmelerinde görünür

Çalışma Sırasında 2FA Gerekirse

Test çalışırken TOTP secret kayıtlı değilse, ekranınıza otomatik modal açılır. Authenticator uygulamanızdaki kodu girin → test devam eder. Tüm süreç canlı log ekranından izlenebilir.

❓ Sıkça Sorulan Sorular

Bu sistem ne işe yarar?

EMS'ten Safir Posta'ya geçiş sürecinde, farklı mail sunucuları ve istemciler arasındaki mesaj iletim kalitesini otomatik olarak test eder. Manuel test yerine her kombinasyonu saniyeler içinde doğrular.

Claude API zorunlu mu?

Evet, analiz motoru Claude API kullanır. API key olmadan testler çalışır ama analiz/değerlendirme (PASS/FAIL kararı) yapılamaz. Key almak ücretsiz deneme ile başlar: console.anthropic.com

Gerçek mail mi gönderiliyor?

Normal modda evet — gerçek SMTP üzerinden gerçek mesajlar gönderilir. Dry Run modunda hayır — sadece bağlantı testi yapılır, mail gönderilmez.

CSV dosyasında tanımlanan sayı kadar. Varsayılan checklist'te 10 kombinasyon × 9 senaryo = 90 test bulunur. Ek kombinasyonlar CSV'ye satır ekleyerek oluşturulabilir.

S/MIME neden devre dışı?

Kurumda dijital imza altyapısı (PKI/CA) henüz aktif olmadığından S/MIME senaryosu otomatik atlanır. Altyapı hazır olduğunda config.yaml'a smime_cert_path ve smime_key_path eklenerek aktif edilir.

Tek bir senaryo ortalama 20-30 saniye (gönderim + bekleme + analiz). Tam bir kombinasyon (9 senaryo) ~4-5 dakika. Tüm 10 kombinasyon ~40-50 dakika sürer.

Rapor dosyaları nerede?

reports/ klasöründe. Her çalışma test_report.html ve test_results.csv üretir. Web arayüzünden "Raporlar" sekmesinde görüntülenip indirilebilir.

Loglara nereden bakılır?

Üç yoldan: (1) Web arayüzü → Canlı Loglar sekmesi (gerçek zamanlı), (2) logs/automation.log dosyası, (3) CLI'da terminal stdout çıktısı.
